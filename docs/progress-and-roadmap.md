# papdoc 进度与后续计划

更新日期：2026-03-24

## 当前定位

`papdoc` 目前是一个面向数据库论文跟踪与阅读辅助的 Android app 原型，重点服务以下方向：

- 存储引擎
- 查询优化
- OLAP / 分析执行

当前形态已经不是单纯网页，而是：

- 采集脚本负责从多个数据库研究渠道抓取论文元数据并生成中文报告
- Next.js 负责前端交互
- Capacitor 提供 Android 原生壳
- Android 端本地缓存已经切到 SQLite

## 已完成进度

### 1. 论文采集与报告生成

已经接入的来源：

- arXiv `cs.DB`
- `PVLDB`
- `PACMMOD / SIGMOD`
- `ICDE`
- `CIDR`
- `DBLP` 元数据补全

已经实现：

- 生成中文 Markdown 报告
- 生成结构化 JSON 数据
- `PVLDB` 优先使用 `vldb.org/pvldb/...pdf` 官方 PDF 链接
- 报告按数据库内核关注方向做初步分类

当前主要输出：

- `reports/db-research-digest.md`
- `reports/archive/`
- `reports/data/latest.json`
- `reports/data/papers/*.json`

### 4. 论文总结能力

已经完成：

- 点击论文后进入独立详情页
- 详情页支持生成单篇论文 AI 总结
- 总结结构固定为：
  - 一句话结论

  - 核心方法 / 系统设计
  - 实验与结果
  - 对数据库内核的启发
  - 是否值得精读
  - 关键术语
- PDF 下载失败时支持手动上传 PDF
- Android 端 PDF 下载与模型请求已加入原生网络兜底

### 5. 关键术语交互

已经完成：

- “关键术语”已做成可点击
- 点击术语后会调用用户配置的大语言模型
- 输出内容包含：
  - 数据库科学范畴内的术语解释
  - 该术语在当前论文中的上下文
  - 该术语为什么对理解论文重要
- 术语解释结果也会缓存到本地

### 6. 本地缓存

已经完成：

- Web 环境保留 IndexedDB 回退
- Android 原生环境改为 SQLite 缓存
- 已缓存的论文会在列表页和详情页显示 `已总结`
- 清空缓存时会同时清空总结缓存和术语解释缓存

当前 SQLite 缓存内容：

- 论文总结缓存
- 术语解释缓存

## 当前已知限制

### 1. SQLite 插件存在一个构建修补步骤

`@capacitor-community/sqlite` 当前接入版本为 `7.0.3`，但其 Android 子工程默认写死了另一套 AGP 版本。为了与本项目当前 Android 构建链兼容，项目里增加了一个 `postinstall` 修补脚本：

- `scripts/patch_sqlite_gradle.py`

当前做法是稳定可用的，但属于工程兼容层，不是长期最理想方案。

### 2. 原生端旧缓存未自动迁移

从 IndexedDB 切到 SQLite 后：

- Android 原生端会以 SQLite 作为新的缓存主存储
- 之前已经写进 IndexedDB 的旧缓存不会自动迁移到 SQLite

### 3. 报告排序仍然偏启发式

目前论文重要性排序主要基于：

- 来源渠道
- 摘要关键词
- 与目标方向的匹配程度

这还不是严格的“研究助手级别”排序，后续还需要继续提高精度。

### 4. 还没有真正的 PDF 阅读器

当前详情页已经为 PDF 阅读区预留了结构，但还没有接入真正的 PDF 阅读组件或原生阅读体验。

## 下一阶段计划

### P1. 接入 PDF 阅读器

目标：

- 在详情页加入真正的 PDF 阅读区
- 支持左侧阅读、右侧 AI 总结/术语解释
- 支持在阅读过程中继续点击术语或触发新的分析动作

建议方向：

- 优先在现有详情页内嵌阅读视图
- 保持后续可替换为更强的原生 PDF 能力

### P2. 做术语与段落级联动

目标：

- 让术语解释可以直接关联到论文中的位置或片段
- 支持“这个术语在文中第一次出现在哪”
- 支持“引用本论文相关段落做上下文总结”

### P3. 完善本地知识与缓存体系

目标：

- 为 SQLite 缓存加版本管理
- 加入缓存迁移逻辑
- 明确区分：
  - 论文总结缓存
  - 术语解释缓存
  - 论文全文缓存
  - 未来的阅读标注缓存

### P4. 强化研究工作流

目标：

- 支持标记“待读 / 已读 / 精读”
- 支持收藏论文
- 支持个人笔记
- 支持将总结和笔记导出

### P5. 提高论文筛选与推荐质量

目标：

- 改进论文分类
- 改进重要性排序
- 增加更符合数据库研究者视角的推荐逻辑
- 更突出存储引擎、查询优化、OLAP 论文

### P6. 完善自动化采集

目标：

- 提高多来源采集稳定性
- 减少噪音论文
- 让中文周报/月报更适合直接阅读
- 为后续“批量 AI digest”预留接口

## 建议的后续开发顺序

建议按下面顺序推进：

1. PDF 阅读器
2. 段落级上下文分析
3. SQLite 缓存版本化与迁移
4. 个人阅读状态 / 收藏 / 笔记
5. 推荐与排序增强
6. 更强的自动化采集与批量 digest

## 备注

当前仓库已经具备继续迭代的基础条件：

- Android app 可以持续安装验证
- 本地模型配置链路已通
- 本地缓存已具备原生存储能力
- 详情页结构已经适合继续扩展阅读工具

后续如果继续开发，建议优先把 `详情页` 作为核心工作台来演进，而不是再回到“列表弹层式”的交互。
